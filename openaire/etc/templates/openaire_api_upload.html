{{config.CFG_SITE_NAME}} Upload API v1.0

The purpose of this document is to explain how to upload via our RESTful API. Before you will be able to use our API you must create an account and generate an API key under your profile.

Audience

This document and the API is meant for use by a technical audience. You will need a working knowledge of programming and the HTTP protocol.

Overview

The API is implemented in RESTful style. The end point for v1 you are going to use is:

  • https://zenodo.org/api/deposit/

At the moment there are two resources:

  • /depositions/
  • /depositions/:uuid/
Characters
UTF8 encoding
Security
All the traffic goes via HTTPS - please remember we do not support bare HTTP. All the requests to this system require API key authentication. Each key have some constraints associated with it:

API Reference

Resources GET depositions - List ongoing and submitted depositions POST depositions - Create a new deposition GET depositions/:uuid - Get deposition PUT depositions/:uuid - Update deposition metadata DELETE depositions/:uuid - Delete deposition GET depositions/:uuid/files - List deposition files POST depositions/:uuid/files - Create a new file GET depositions/:uuid/files/:id - Get file metadata PUT depositions/:uuid/files/:id - Replace file metadata DELETE depositions/:uuid/files/:id - Delete file
